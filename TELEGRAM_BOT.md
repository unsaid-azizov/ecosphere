# Telegram Notification Bot

## Описание
Бот для отправки уведомлений всем пользователям EcoSphere о важных событиях в системе.

## Возможности
- ✅ Авторизация через существующие учетные данные (email/password)
- ✅ Доступ для всех пользователей (обычных, менеджеров, администраторов)
- ✅ **Для обычных пользователей**: Уведомления об изменении статуса их заказов
- ✅ **Для администраторов и менеджеров**:
  - Уведомления о новых зарегистрированных пользователях
  - Детальные уведомления о новых заказах (со списком товаров)
  - Уведомления об изменении статусов всех заказов

### 2. Запуск бота
Бот автоматически запускается вместе с Next.js приложением при выполнении:
```bash

```

Или можно запустить бот отдельно:
```bash
npm run telegram-bot
```

## Использование

### Для всех пользователей:

1. **Найдите бота в Telegram**
   - Откройте Telegram
   - Найдите вашего бота по имени или ссылке

2. **Начните диалог**
   ```
   /start
   ```

3. **Авторизуйтесь**
   ```
   /login your@email.com yourpassword
   ```
   Используйте те же учетные данные, что и для входа на сайт.

4. **Проверьте статус**
   ```
   /status
   ```

5. **Получайте уведомления!**

   **Обычные пользователи** получают:
   - 🔔 Уведомления об изменении статуса своих заказов

   **Администраторы и менеджеры** получают:
   - 🆕 Новых зарегистрированных пользователях
   - 📦 Новых заказах (с подробной информацией о товарах)
   - 🔄 Изменениях статусов всех заказов

## Команды бота

- `/start` - Начать работу с ботом
- `/login <email> <password>` - Войти в систему
- `/status` - Проверить статус подключения
- `/logout` - Выйти из системы (отключить уведомления)
- `/help` - Показать справку

## Безопасность
- ✅ Доступ для всех зарегистрированных пользователей
- ✅ Каждый пользователь видит только свои уведомления (кроме администраторов)
- ✅ Пароли хранятся в зашифрованном виде
- ✅ Авторизация через bcrypt
- ✅ Сессии хранятся в памяти (не в базе данных)

## Технические детали

### Библиотеки
- `node-telegram-bot-api` - API для работы с Telegram Bot
- `bcryptjs` - Проверка паролей
- `@prisma/client` - Работа с базой данных

### Архитектура
- Бот запускается автоматически при старте Next.js
- Авторизованные чаты хранятся в памяти (`Map<chatId, userData>`)
- При перезапуске сервера пользователям нужно авторизоваться заново
- Уведомления отправляются асинхронно, не блокируют основные операции

### Расширение функционала
Для добавления новых уведомлений используйте функции из `src/lib/telegram-bot.ts`:
- `notifyNewUser()` - Новый пользователь
- `notifyNewOrder()` - Новый заказ
- `notifyOrderStatusChange()` - Изменение статуса
- `notifyCriticalEvent()` - Критическое событие
- `sendNotificationToAdmins()` - Произвольное сообщение

## Troubleshooting

### Бот не отвечает
1. Проверьте, что токен правильно указан в `.env`
2. Проверьте, что Next.js сервер запущен
3. Проверьте логи в консоли: должно быть `✅ Telegram bot started`

### Не могу авторизоваться
1. Убедитесь, что используете правильный email и пароль
2. Проверьте, что ваша роль - ADMIN или MANAGER
3. Проверьте, что пользователь существует в базе данных

### Не приходят уведомления
1. Убедитесь, что вы авторизованы (выполните `/status`)
2. Проверьте, что сервер не был перезапущен после авторизации
3. Проверьте логи сервера на ошибки отправки сообщений
